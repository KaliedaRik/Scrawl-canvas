Scrawl-canvas v8 - Modules test 006

Animation observer; Scrollytelling

This demo covers/tests progressive enhancement, responsive canvases and scrollytelling - page-based infographics which unfold as the user scrolls down the page

  • Progressive enhancement - the page should display and tell its story when the user has disabled Javascript in their browser. All key data should be accessible to users using, for instance, screen readers. And users should be able to tab through the page contents even when Javascript is enabled - they should not be held hostage to the scroll!
  • Responsive - the infographic elements should tell their animated stories on both mobile and desktop devices, including user interaction.
  • The scrollytelling (an ugly word) experience should be as smooth as possible. It should also respect user preferences such as prefers-reduced-motion and prefers-color-scheme.

Touch test: in theory the user should be able to scroll through the page and canvases should run/animate as/when they come into view and move through the viewport. The UX on desktop seems okay; on touch-based devices the results - especially for the full-width infographic canvases - are not satisfactory. We need to think of a different approach if we want SC to be useful for scroll-based, animation-heavy storytelling.

Canvas 1. The first canvas - fixed-dimensions, unresponsive - stays positioned in the flow of the page, moving with the copy above and below it. The canvas displays its animationObserver position within the viewport. The graphic text in the canvas will size responsively alongside the surrounding article's text size (as set in CSS)

This browser environment has disabled Javascript

Test purpose

  • Create several canvases, both user-controlled responsive and banner.
  • Use each canvas wrapper's `here` object's `inViewportTop`, `inViewportCenter` and `inViewportBase` attributes to control their displays.
  • Check that each canvas element's related animation object only runs when the canvas is visible in the viewport.
